## Backlog Prioritization Rules

### P0 (Critical) — Must resolve before proceeding
- Slot is EMPTY and blocks implementation
- Slot has CONFLICTING evidence that affects core functionality
- Hypothesis confidence is LOW and impact if wrong is CRITICAL

### P1 (Important) — Should resolve before deployment
- Slot is VAGUE and affects deployment decisions
- Slot is EMPTY but has reasonable fallback
- Hypothesis confidence is MEDIUM and impact if wrong is SIGNIFICANT

### P2 (Nice to Have) — Can defer
- Slot is VAGUE but doesn't affect core functionality
- Slot is EMPTY but impact is MINOR
- Hypothesis confidence is HIGH enough to proceed
